# 桌宠 2.0

桌宠 2.0 是一个基于 Electron、React 和 Vite 的本地优先智能桌宠原型。项目目标不是做普通陪聊窗口，而是做一个能理解工作节奏、反馈状态、提供情绪价值，并可逐步养成性格的桌面伙伴。

## 项目定位

桌宠 2.0 的核心方向是：

- 以情感陪伴为最高定位，而不是单纯的 AI 聊天工具。
- 通过本地事件、工作状态、番茄钟、备忘录、天气和用户互动来触发桌宠行为。
- 接入 DeepSeek 大模型，让桌宠的动作和反馈不再完全依赖固定规则。
- 保留轻量、低打扰、低资源占用的桌面体验。
- 支持自定义角色、动作素材库和可训练的桌宠性格。

## 当前能力

- 桌宠悬浮窗：独立宠物前端，支持透明桌面展示。
- 控制面板：用于配置角色、状态、动作素材、DeepSeek、记忆和辅助功能。
- 托盘入口：支持后台运行和快速打开控制面板。
- Mini Mode：保留轻量化显示模式。
- DeepSeek 集成：支持事件决策、动作结构整理和智能反馈。
- 记忆系统：记录事件统计、偏好信号、关系指标和近期洞察。
- 动作素材库：按待机、开心、委屈、睡觉、工作事件、天气、番茄钟等场景管理素材。
- 素材处理流水线：支持透明帧、GIF 预览、动作帧目录和 spritesheet 输出。
- 单图自动动图：选择一张角色参考图后，可一键生成动作条、GIF 预览和 spritesheet，作为后续接入真实 image gen 的产品闭环。
- 番茄钟与备忘录：作为桌宠主动提醒和情绪反馈的工作辅助场景。

## 产品理念

桌宠应该像一个逐渐了解用户习惯的伙伴：

- 用户开始工作时，它能感知节奏变化。
- 用户长时间离开时，它能进入等待或睡眠状态。
- 用户完成任务时，它能给出情绪化反馈。
- 用户长时间没有互动时，它能用更有存在感的动作提醒自己。
- 用户持续使用后，它能根据记忆逐渐形成偏好和性格。

项目坚持本地优先原则：不读取键盘具体输入，不截图，不做高风险监控，尽量减少杀毒软件误报和隐私风险。

## 技术栈

- Electron
- React
- Vite
- JavaScript
- Python 素材处理脚本
- DeepSeek Chat Completions API

## 本地运行

安装依赖：

```bash
npm install
```

开发运行：

```bash
npm run dev
```

生产构建：

```bash
npm run build
```

完整验证：

```bash
npm run verify:pet-sprite
```

## DeepSeek 配置

项目默认通过环境变量读取 DeepSeek API Key：

```bash
DEEPSEEK_API_KEY=你的_key
```

当前请求体使用 `deepseek-v4-flash`，并保留 reasoning/thinking 配置验证脚本。

## 目录结构

```text
src/main/       Electron 主进程、状态管理、系统能力和 DeepSeek 调用
src/panel/      控制面板前端
src/pet/        桌宠显示前端
src/shared/     行为引擎、记忆系统、动作素材结构和播放逻辑
scripts/        素材处理与验证脚本
docs/           产品设计和实现计划文档
```

## 当前阶段

这是一个 MVP 到产品化过渡阶段的项目。当前重点是打通桌宠展示、控制面板、动作素材、AI 决策、记忆系统和日常工作辅助。后续重点会放在：

- 首次使用流程
- 接入真实 image gen API，替换当前本地动作条生成器
- 事件触发解释与反馈调节
- 记忆系统可视化和用户确认机制
- 更稳定的角色包格式
- 更自然的动画质量

## 隐私边界

项目设计上避免以下行为：

- 不读取用户键盘输入内容。
- 不截屏。
- 不采集敏感个人内容。
- 不上传本地日志，除非用户主动配置相关能力。
